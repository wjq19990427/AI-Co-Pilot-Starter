# 如何编写 L2 契约文档

> L2 契约是模块公开 API 的唯一权威来源。实现工手改代码前必须读这里，
> 改了公开签名/副作用语义后必须同步更新这里。

## 文件命名

每个代码层一个文件：`docs/api/{layer}.md`

## 文件结构模板

```markdown
# {layer}/ — {层职责描述}

> {一句话说明边界规则，如：只消费下层，不实现业务逻辑}

## 边界规则

- {规则1}
- {规则2}

## 模块清单

| 文件 | 职责 | 契约状态 |
|------|------|----------|
| `foo.py` | {职责} | ✅ / 🚧 待填 |

---

## foo.py

> {模块一句话描述}

### `function_name(param1: Type, param2: Type) -> ReturnType`

- **入参**：
  - `param1`：{说明}
  - `param2`：{说明}
- **返回**：{说明}
- **副作用**：{DB 写入、文件操作、session_state 变更等}
- **已知陷阱**：{容易踩的坑}
```

## 写什么，不写什么

**写**：
- 公开函数/类的签名、入参约束、返回结构
- 非显而易见的副作用（写 DB、触发 hook、改全局状态）
- 调用方需要知道的约束（必须在 X 之前调用、不能在 Y 上下文中用）
- 已知陷阱（踩过坑的地方）

**不写**：
- 实现细节（那是 L3 源代码的事）
- 注释掉的代码或临时 TODO
- 明显从函数名就能读出来的内容

## 更新时机

- 新增公开函数/类 → 补充对应节
- 修改函数签名 → 更新签名和入参说明
- 改变副作用语义（如原来不写 DB 现在写了）→ 更新副作用节
- 纯内部实现重构，签名和语义不变 → 不需要更新
